## 📚 이번 주 학습한 것

### 1. DOM & 비동기 및 React 입문
- **핵심 개념**: DOM 제어, 비동기 데이터 처리(async/await, fetch), React 컴포넌트, JSX, `useState` 및 `useEffect`
- **내가 이해한 방식**:
  - JS로 DOM을 직접 조작하던 방식에서 벗어나, React가 선언적으로 UI를 관리하고 가상 DOM을 통해 최적화하는 방식을 학습했다.
  - `useState`로 컴포넌트 내부 상태를 제어하고, API 호출이나 외부 효과 처리를 위해 `useEffect`를 사용해 컴포넌트 렌더링 시점을 제어하는 법을 배웠다.

### 2. React 구조와 상태 관리
- **핵심 개념**: 컴포넌트 분리 전략(SRP), Props(단방향 데이터 흐름), Lifting State Up(상태 끌어올리기), Custom Hook
- **내가 이해한 방식**:
  - 하나의 컴포넌트에 너무 많은 책임을 주지 않고 역할별로 컴포넌트를 분리하여 재사용성과 유지보수성을 높인다.
  - React는 단방향 바인딩이므로 형제 컴포넌트 간 데이터 공유가 필요할 경우 공통 부모로 상태를 끌어올려(`Lifting State Up`) 해결해야 한다.
  - 비동기 로직이나 상태 관리 패턴이 반복될 때는 `use`로 시작하는 Custom Hook으로 분리하여 UI와 비즈니스 로직을 깔끔하게 나눌 수 있다.

### 3. Tailwind CSS 스타일링
- **핵심 개념**: Utility-First CSS, 반응형 설계(sm, md, lg), hover/focus 상태 제어, Flexbox 레이아웃
- **내가 이해한 방식**:
  - 별도의 CSS 파일 정의나 클래스 네이밍 고민 없이, className 안에 미리 지정된 유틸리티 클래스들을 조합해 빠르게 스타일링할 수 있다.
  - 모바일 퍼스트 반응형 접두사와 Flexbox(`justify-between`, `items-center` 등)를 사용해 레이아웃을 효율적으로 제어할 수 있다.

## 🧱 막혔던 지점 & 해결 과정
- **문제 상황**: 이번 주차 학습 도중 엘리스(Elice) 학습 사이트 서버가 계속 불안정하여 동영상 강의가 끊기거나 실습 서버 연결이 원활하지 않았다. 이로 인해 코드 실습을 꼼꼼하게 다 해보지 못했고, 월요일 1시 제출 마감 시간에 쫓기게 되었다.
- **시도한 방법**: 일단 제출 마감 요건을 지키기 위해 실습 파트 및 이론의 세부 내용을 빠르게 훑어보고(스킵하며 핵심 위주 파악) 학습일지 초안을 먼저 작성하여 기한 내 제출에 집중했다.
- **최종 해결 / 참고 자료**: 우선 제출을 처리한 후, 서버가 안정되는 시점이나 다음 주 초에 시간을 따로 내어 미진했던 React 상태 관리 및 실습 코드 작성 부분을 복습하여 재학습할 예정이다.
