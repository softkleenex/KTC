## 1️⃣ 파이썬 기초와 자료형

**변수와 자료형** — 데이터에 이름 붙이기

```python
# 변수 = 데이터에 붙이는 이름표
user_name = "카카오"          # str: 문자열 — 채팅 메시지, 이름, AI 응답
age = 23                     # int: 정수  — 나이, 개수
score = 0.95                 # float: 실수 — 확률, 점수
is_active = True             # bool: 참/거짓 — 로그인 여부, 활성화 상태

chat_history = ["안녕", "날씨?", "감사"]   # list: 순서 있는 목록 — 채팅 기록
user = {"name": "카카오", "age": 23}       # dict: 이름표 붙은 데이터 — 사용자 프로필
```

### 리스트 vs 딕셔너리

|**구분**|**리스트 (List) [ ]**|**딕셔너리 (Dictionary) { }**|
|---|---|---|
|**데이터 구조**|값(Value)들의 단순 나열|키(Key)와 값(Value)|
|**데이터 찾기 (접근)**|**위치(인덱스)**로 찾음 (n번째)|**이름(키, Key)**으로 찾음|
|**순서의 중요성**|순서가 매우 중요함|순서보다 데이터의 '이름(키)'이 더 중요함|
|**언제 쓰나요?**|**순서대로** 데이터를 모아둘 때||
|_(ex. 장바구니 목록, 대기열 순번)_|데이터에 **이름을 붙여** 묶어둘 때||
|_(ex. 유저 정보, 연락처, 설정값)_|||
|**코드 예시**|items = ["사과", "바나나", "포도"]||
|print(items[0]) _# 사과 출력_|user = {"이름": "카카오", "나이": 20}||
|print(user["이름"]) _# 카카오 출력_|||

**데이터 가공과 처리**

```python
# 아래에서 활용할 예시 데이터
chat_history = ["안녕", "날씨?", "감사"]
user = {"name": "카카오", "age": 23}
user_name = "카카오"

# str — 텍스트 깔끔하게 다듬기
print("  hello  ".strip())              # "hello" (양옆 공백 제거)
print("hello world".split())            # ["hello", "world"] (단어 쪼개기)
print(f"안녕, {user_name}님!")           # f-string: 변수를 문장 안에 쏙 넣기

# list — 목록(데이터) 추가하고 찾기
chat_history.append("또 올게요")         # 리스트 맨 뒤에 데이터 추가
print(chat_history[0])                   # "안녕" (첫 번째 데이터 꺼내기)
print(chat_history[-1])                  # "또 올게요" (가장 마지막 데이터 꺼내기)

# dict — 이름표(Key)로 정보 꺼내기
print(user["name"])                      # "카카오"
print(user.get("email", "없음"))         # 키가 없어도 에러 없이 "없음" 반환
```

**입출력 & 형변환**

```python
user_input = input("나이를 입력하세요: ")  # input() -> 항상 문자열(str) 자료형으로 반환
age = int(user_input)                 # str → int 변환
print(f"내년엔 {age + 1}살이에요")

# str(), float()도 같은 방식
```

> **🚨 주의사항** `b = a` (리스트)는 복사가 아니라 같은 리스트를 함께 가리키는 것. (편집 권한 있는 공유 문서 전달한 것과 마찬가지인 상황) 사본 만들기 개념의 독립 복사는 `b = a.copy()`.

## 2️⃣ 조건문과 반복문

> ex) 사용자가 VIP인지 일반인지에 따라 다른 응답을 주거나(조건문), 100개의 채팅 기록을 하나씩 처리하거나(반복문)

**조건문 — 상황에 따라 다른 동작**

```python
# 비교: ==, !=, >, <, >=, <=
# 논리: and, or, not, in

user_type = "vip"
message_count = 150

# 먼저 VIP인지 일반인지 확인
if user_type == "vip":
    
    # 메시지 개수에 따라 추가 분류
    if message_count < 200:
        print("VIP 프리미엄 응답")
    else:
        print("VIP이지만 한도 초과")

else:
    print("일반 응답")
```

**반복문 — 같은 작업 반복**

```python
# for — 리스트를 순서대로 순회
messages = ["안녕", "날씨?", "감사"]
for i, msg in enumerate(messages):
    print(f"{i+1}번: {msg}")

# 출력 결과
# 1번: 안녕
# 2번: 날씨?
# 3번: 감사

# while — 조건이 True인 동안 반복 (카운터 빠뜨리면 무한 루프!)
retry = 0
while retry < 3:
    print(f"재시도 {retry + 1}회")
    retry += 1
    
# 출력 결과
# 재시도 1회
#재시도 2회
#재시도 3회
```

**List Comprehension — 한 줄로 리스트 만들기**

```python
scores = [85, 42, 91, 67, 78]

passed = [s for s in scores if s >= 80]    # 80점 이상만 → [85, 91]
labels = [f"{s}점" for s in scores]         # 문자열로 변환 → ["85점", ...]

print(passed)
print(labels)
```

### List Comprehension 장단점

|구분|👍 장점 (Pros)|👎 단점 (Cons)|
|---|---|---|
|코드 가독성|• 여러 줄의 for, if 문을 단 한 줄로 압축 가능||
|• '리스트를 만든다'는 목적이 한눈에 보임|• 로직이 중첩될 경우 암호문처럼 가독성 최악||
|• 코드를 해석하는 데 오히려 시간이 더 걸림|||
|실행 성능|• 파이썬 내부 최적화로 일반 for문보다 속도가 빠름|• (성능 자체의 단점은 없음)|
|디버깅 및 유지보수|• 단순한 데이터 가공 시 코드가 매우 깔끔해짐|• 코드가 한 줄이라 중간에 print()로 확인 불가|
|• 에러 발생 시 정확한 위치를 추적하기 힘듦|||

## 3️⃣ 함수, 모듈, 파일 입출력

> 코드의 재사용성을 높이는 방법

**함수 (def)**

```python
def greet(name, role="user"):     # role에 기본값 설정
    return f"안녕하세요, {name}({role})님!"

print(greet("카카오"))             # 안녕하세요, 카카오(user)님!
print(greet("관리자", "admin"))    # 안녕하세요, 관리자(admin)님!
```

**모듈 import & 파일 입출력**

```python
# [import vs from 차이점]
import json                   # 모듈 전체 가져오기 -> json.기능() 형태로 씀
from datetime import datetime # 모듈에서 특정 함수/변수만 가져오기 -> 기능() 바로 씀

# 1. 파일 쓰기 (with를 쓰면 작업 후 파일이 자동으로 안전하게 닫힘)
data = {"user": "카카오", "time": str(datetime.now())}  # datetime 바로 사용

with open("chat.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False) # json 모듈 안의 dump 기능 사용

# 2. 파일 읽기
with open("chat.json", "r", encoding="utf-8") as f:
    saved_data = json.load(f)
    print(saved_data["time"])  # 방금 저장한 시간 출력
```

## 4️⃣ 클래스·예외·디버깅

**클래스 — 데이터 + 기능을 하나로 묶기**

> 수백만 명이 동시에 쓰는 서비스를 만든다고 가정해 봅시다. 유저마다 이름, 채팅 기록, 구독 등급이 다릅니다. 이를 변수로 따로따로 관리하면 금세 뒤엉키고 맙니다.
> 
> 클래스는 **"유저 한 명 분의 틀"**을 만들어, 필요한 만큼 인스턴스를 찍어내는 방법입니다.

- 클래스와 인스턴스 비유로 이해하기
    - **클래스 (Class) = 로봇 설계도**
        - "이 로봇은 '이름'을 가지고 태어나고, '인사'를 할 수 있다"라고 적어둔 종이 문서
        - 아직 실체는 없음
    - **인스턴스 (Instance) = 공장에서 찍어낸 실제 로봇**
        - 설계도를 보고 실제로 조립해 낸 로봇 기계 1대
        - 로봇 A와 로봇 B를 만들면, 둘은 완전히 별개의 기계

```python
# ① 챗봇 설계도(Class) 작성하기
class ChatBot:
    # __init__ : 챗봇이 딱 생성되는 순간(앱을 처음 켰을 때) 무조건 1번 실행되는 초기 세팅
    def __init__(self, name):
        self.name = name       # "내(self) 이름은 전달받은 이름으로 세팅해라"
        self.history = []      # "내(self) 대화창은 빈 리스트([])로 깨끗하게 시작해라"

    # 메서드 : 이 챗봇이 할 수 있는 '행동' (대화하기)
    def chat(self, msg):
        self.history.append(msg) # 내가 받은 메시지를 내(self) 대화창에 저장해라
        return f"[{self.name}] 대화 기록이 {len(self.history)}개 있습니다."

# ② 설계도를 보고 실제 챗봇(Instance) 2개 만들어내기
# 이때 __init__이 자동 실행되면서 이름과 빈 대화창이 각각 세팅됨!
bot_1 = ChatBot("카카오봇")  
bot_2 = ChatBot("마카오봇")  

# ③ 만들어진 챗봇과 대화해보기 (둘은 대화가 안 섞임)
print(bot_1.chat("안녕?"))      # 출력: [카카오봇] 대화 기록이 1개 있습니다.
print(bot_1.chat("오늘 날씨?")) # 출력: [카카오봇] 대화 기록이 2개 있습니다.

# 마카오봇은 카카오봇과 완전히 독립된 챗봇이므로 대화 기록이 0개부터 시작함!
print(bot_2.chat("반가워!"))    # 출력: [마카오봇] 대화 기록이 1개 있습니다.
```

💡 **[Day 1 Preview]** 만약 오늘 만든 ChatBot 클래스에 기능이 20개, 100개로 늘어나면 어떻게 될까요? 1주차 Day 1 본 수업에서는 이렇게 복잡해진 객체를 어떻게 효율적으로 구조화하고 확장할지(@dataclass, 캡슐화, 상속)를 심화해서 배웁니다.

**예외처리 (try-except) — 에어백 같은 역할**

```python
def parse_age(value):
    try:
        return int(value)
    except ValueError:
        print(f"'{value}'는 숫자가 아닙니다.")
        return None

print(parse_age("23"))      # 23
print(parse_age("hello"))   # 오류 메시지 출력 후 None 반환 (프로그램 계속 실행)
```

## 5️⃣ 개발환경 세팅 및 Git 기초

> Git 없이 코딩하는 건 저장 없이 게임하는 것과 같다. Git을 활용하면 실수해도 되돌아갈 수 있고, 팀원과 코드를 합칠 수 있다.

**Git vs GitHub**

|구분|역할|비유|
|---|---|---|
|**Git**|내 컴퓨터에서 코드 변경 이력을 기록하는 도구|개인 일기장|
|**GitHub**|그 기록을 클라우드에 올려 팀원과 공유하는 서비스|팀 공유 드라이브|

**기본 명령어 흐름**

```bash
git init                           # 폴더를 Git으로 관리 시작
git status                         # 현재 변경사항 확인
git add .                          # 변경 파일을 커밋 대기열에 올리기
git commit -m "feat: 로그인 기능"     # 이력에 저장 (= 게임 세이브)
git push origin main               # GitHub에 업로드
git pull origin main               # 팀원 변경사항 내려받기
git log --oneline                  # 커밋 이력 한 줄 요약
```

**🔥 주의사항**

- (API 키, 비밀번호 등이 있는) `.env` 파일은 GitHub에 올리면 안 됩니다.
    - 집 주소랑 비밀번호를 에브리타임 게시판에 올려놓는 것과 같습니다.
- 대신 `.gitignore` 이라는 파일을 만들고, 그 파일 안에 GitHub에 올리기 싫은 파일이나 폴더 이름 등을 적어주세요.
    - 만약 실제 키가 유출됐다면? 즉시 해당 키를 삭제하고 재발급받아야 합니다.
