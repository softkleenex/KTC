## 🤔 FastAPI란?

<aside> 📌

**개념**

Python으로 만드는 **고성능 웹 API 프레임워크**

- [타입 힌트 기반으로 **자동 유효성 검증**](https://fastapi.tiangolo.com/ko/python-types/#edit-it)
- **Swagger UI 자동 생성**
- **비동기(async)** 지원으로 빠른 처리 </aside>

<aside> 🍕

**비유**

FastAPI = **배달 앱의 주문 접수 창구**

고객(클라이언트)이 "피자 주문할게요"라고 하면 창구(FastAPI)가 요청을 받아서 주방(AI 모델/DB)에 전달하고, 결과를 다시 고객에게 돌려주는 과정

</aside>

**가장 기본적인 FastAPI 서버**

```python
# main.py
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def root():
    return {"message": "Hello, FastAPI!"}
```

```bash
# 서버 실행
uvicorn main:app --reload

# 출력:
# INFO:     Uvicorn running on <http://127.0.0.1:8000> (Press CTRL+C to quit)
```

<aside> 💡

`--reload`: 코드 수정 시 서버가 자동으로 재시작(개발 중 사용)

- 운영 환경에서는 비추천하는 이유
    - 리소스 감시 비용: 코드가 변경되었는지 실시간으로 계속 감시하느라 CPU와 메모리 자원을 불필요하게 낭비
    - 서비스 중단 위험: 운영 중 예기치 않게 파일이 변경되면 서버가 자동으로 재시작되어 사용자의 접속이 끊길 수 있음 </aside>

## 🛣️ 라우팅 정의 → 백엔드 서비스 진입점 구현 실습

### 라우팅이란?

<aside> 📌

**개념**

어떤 URL로 들어오는 요청을 어떤 함수가 처리할지 **연결**해주는 것

ex) `GET /users/123` 요청 → `get_user(user_id=123)` 함수 실행

---

- **서비스 사용자 입장**: 카카오 앱에서 "채팅 목록" 탭을 누르면 `/api/chats`(예시) 요청 → 라우팅이 그 요청을 알맞은 함수로 연결
- **개발자 입장**: 기능마다 URL을 분리해서 **코드를 역할별로 관리** 가능 </aside>

```jsx
기본 URL (Host & 경로)          경로 파라미터 (Path)      쿼리 파라미터 (Query)
┌─────────────────────────┐    ┌─────────────────┐    ┌─────────────────────────────┐
<http://127.0.0.1:8000/users>       /123/messages         ?page=1&sort=latest&limit=10
└─────────────────────────┘    └─────────────────┘    └─────────────────────────────┘
             │                          │                            │
      [어디로 가야 하나요?]        [누구를 찾으시나요?]          [어떻게 보여줄까요?]
       서버의 기본 주소와         "123번 유저의 메시지"         "1페이지 데이터를 최신순으로
       큰 카테고리(도메인)         딱 찝어서 식별할 때           10개만 필터링해서 보여줘"
```

```jsx

uvicorn 01_router:app --reload

# 출력:
# INFO:     Uvicorn running on <http://127.0.0.1:8000> (Press CTRL+C to quit)
```

### GET 라우팅 기본

```python

users_database = [
    {"id": 1, "name": "카카오"},
    {"id": 2, "name": "테크캠"},
    {"id": 3, "name": "엘리스"},
    {"id": 4, "name": "tech"},
    {"id": 5, "name": "python"},
]

from fastapi import FastAPI

app = FastAPI()

# GET /users → 전체 유저 목록 조회
@app.get("/users")
def get_users():
    return users_database
```

### [쿼리 파라미터 (Query Parameter)](https://fastapi.tiangolo.com/ko/tutorial/query-params/)

URL 끝에 물음표(?)를 붙이고 키=값 형태로 추가 옵션을 주는 방식입니다. 주로 데이터의 **검색, 필터링, 페이지 나누기(페이징)**에 사용합니다. (예: "유저 목록 주는데, 10명 건너뛰고 5명만 보여줘")

```python
# GET /users?skip=1&limit=2 → 페이지네이션
@app.get("/users/filtered")
def get_filtered_users(skip: int = 0, limit: int = 2):
    return users_database[skip : skip + limit]
```

<aside> ✅

- [http://127.0.0.1:8000/users](http://127.0.0.1:8000/users) → 유저 1~5
- [http://127.0.0.1:8000/users/filtered?skip=1&limit=2](http://127.0.0.1:8000/users/filtered?skip=1&limit=2)  → 유저 2~3만 나옴 </aside>

### POST 라우팅 (데이터 받기)

<aside> 💡

**잠깐! Pydantic과 BaseModel이 뭔가요?**

- 내가 만든 데이터 클래스에 **자동 검사 기능**과 **자동 타입 변환 기능**을 붙이고 싶을 때 쓰는 것이 **Pydantic**
- 그 기능을 쓰기 위해 빌려오는 기초 설계도가 **BaseModel**

---

- 비유적인 설명
    
    **피단틱(Pydantic)** 라이브러리 전체가 큰 의미의 **'검문소 시스템'**이라면, **베이스모델(BaseModel)**은 그 검문소에서 사용하는 **'심사 기준 서식'**입니다.
    

</aside>

```python
class UserCreateRequest(BaseModel):
    id: int
    name: str

@app.post("/create_users")
def create_user(request: UserCreateRequest):
    # 주소창에서 확인하기 위해, 받은 데이터를 리스트에 넣어줍니다.
    users_database.append(request)

    return {
        "status": "success",
        "id": request.id,
        "name": request.name,
    }
```

<aside> ✅

- [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs) → try it out → id, name 수정 → [http://127.0.0.1:8000/users](http://127.0.0.1:8000/users) </aside>

### HTTPException으로 에러 처리

<aside> 📌

**개념**

데이터가 없거나, 권한이 없는 상황 등에서 HTTP 상태코드와 함께 **의미 있는 에러 메시지**를 반환하는 방법

단순히 `return {"error": "..."}` 하면 상태코드는 200일 수 있습니다. (실제로는 에러)

`HTTPException`을 쓰면 `404`, `403`, `400` 같은 올바른 상태코드를 함께 보낼 수 있습니다.

</aside>

```python
from fastapi import FastAPI, HTTPException

@app.get("/users/{user_id}")
def get_user(user_id: int):
    for user in users_database:
        if user["id"] == user_id:
            return user
            
    raise HTTPException(
        status_code=404, 
        detail=f"유저 {user_id}를 찾을 수 없습니다"
    )
```

<aside> ✅

- [http://127.0.0.1:8000/users/99](http://127.0.0.1:8000/users/99) → {"detail": "세션 99를 찾을 수 없습니다"} </aside>

<aside> 💡

**자주 쓰는 상태코드**

- `400 Bad Request`: 잘못된 요청 (파라미터 오류)
- `401 Unauthorized`: 로그인 필요
- `403 Forbidden`: 권한 없음
- `404 Not Found`: 데이터 없음
- `409 Conflict`: 이미 존재 (중복) </aside>

## ⚡ async def — 비동기 엔드포인트

<aside> 📌

**개념**

FastAPI는 `def`와 `async def` 둘 다 지원합니다.

- `def`: 일반 함수 — 일반적인 DB 쿼리나 `requests` 같은 동기 라이브러리를 쓸 때 사용
- `async def`: 비동기 함수 — `await`를 지원하는 비동기 라이브러리나 API 호출 시 사용 (응답을 기다리는 동안 다른 요청을 처리할 수 있어요)

---

- **서비스 사용자 입장**: 챗봇에 질문했을 때 AI 응답이 오는 동안 다른 사람의 요청도 처리돼요. async 덕분에 한 명이 기다리는 동안 서버가 멈추지 않아요.
- **개발자 입장**: OpenAI, Kakao 같은 AI API를 호출할 때 응답 대기 시간을 낭비하지 않고 다른 요청을 처리할 수 있어요. </aside>

```jsx
# 서버 실행
uvicorn 02_async:app --reload

# 출력:
# INFO:     Uvicorn running on <http://127.0.0.1:8000> (Press CTRL+C to quit)
```

```python
import asyncio
from datetime import datetime
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class ChatRequest(BaseModel):
    message: str

async def call_ai_api(message: str) -> str:
    await asyncio.sleep(3)  # AI API 호출 시뮬레이션 (3초 대기)
    return f"AI 답변: {message}에 대한 응답"

@app.post("/chat")
async def chat(request: ChatRequest):
    # 요청이 서버에 들어온 순간의 시간(초) 기록
    start_time = datetime.now().strftime("%H시 %M분 %S초")
    
    # 3초 동안 대기하며 AI 응답을 받아옵니다.
    response = await call_ai_api(request.message)  
    
    # 처리가 완전히 끝난 순간의 시간 기록
    end_time = datetime.now().strftime("%H시 %M분 %S초")
    
    return {
        "start_time": start_time,
        "end_time": end_time,
        "response": response
    }
```

## 📁 APIRouter로 코드 구조화 _(참고)_

<aside> 📌

**개념**

엔드포인트가 많아지면 `main.py` 하나에 모두 담기 어려워요.

`APIRouter`를 쓰면 기능별로 파일을 나눠서 관리할 수 있습니다.

---

- **개발자 입장**: 팀에서 여러 명이 동시에 개발할 때, 각자 다른 파일([users.py](http://users.py), [chat.py](http://chat.py))을 담당하면 충돌 없이 작업 가능해요. 기능을 추가하거나 삭제할 때도 해당 파일만 건드리면 돼요 </aside>

**폴더 구조**

```
chatbot/
├── main.py              # 진입점 — 라우터를 가져와서 등록
├── routers/
│   ├── __init__.py
│   ├── users.py         # /users 관련 엔드포인트
│   └── chat.py          # /chat 관련 엔드포인트
└── models/
    └── schemas.py       # Pydantic 모델
```

```jsx
# routers/users.py
from fastapi import APIRouter, HTTPException

router = APIRouter(prefix="/users", tags=["users"])
fake_db = {1: {"id": 1, "name": "카카오"}}

@router.get("/")
def get_users():
    return list(fake_db.values())
    # [{"id": 1, "name": "카카오"}]

@router.get("/{user_id}")
def get_user(user_id: int):
    if user_id not in fake_db:
        raise HTTPException(status_code=404, detail="유저를 찾을 수 없습니다")
    return fake_db[user_id]
    # {"id": 1, "name": "카카오"}
```

```python
# routers/chat.py
from fastapi import APIRouter
from pydantic import BaseModel

router = APIRouter(prefix="/chat", tags=["chat"])

class MessageRequest(BaseModel):
    user_id: int
    content: str

@router.post("/messages")
def send_message(request: MessageRequest):
    return {"status": "sent", "content": request.content}
    # {"status": "sent", "content": "안녕하세요"}
```

```jsx
# main.py — 라우터를 가져와서 앱에 등록
from fastapi import FastAPI
from routers import users, chat

app = FastAPI(title="카카오 챗봇 API")

app.include_router(users.router)
app.include_router(chat.router)

# → GET  /users/         (users 라우터)
# → GET  /users/{id}     (users 라우터)
# → POST /chat/messages  (chat 라우터)
```

<aside> 💡

**prefix와 tags**

- `prefix="/users"`: 라우터 내 모든 경로 앞에 자동으로 `/users`가 붙어요
- `tags=["users"]`: Swagger UI에서 같은 태그끼리 묶여서 표시돼요 </aside>

### 📝 실습: 세션 API 만들기

<aside> 📝

**실습 문제**

아래 API 엔드포인트를 `APIRouter`로 구현해보세요!

- `POST /sessions` → 새 채팅 세션 생성. `user_id`와 `bot_name`을 받아서 세션 정보 반환
- `GET /sessions/{session_id}` → 세션 정보 조회 (없으면 404)
- `DELETE /sessions/{session_id}` → 세션 종료 </aside>

```jsx
# 서버 실행
uvicorn practice.01_prac:app --reload

# 출력:
# INFO:     Uvicorn running on <http://127.0.0.1:8000> (Press CTRL+C to quit)
```

```jsx
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel

app = FastAPI()

router = APIRouter(prefix="/sessions", tags=["sessions"])
sessions = {}

class SessionCreate(BaseModel):
    user_id: int
    bot_name: str

@router.post("/")
def create_session(request: SessionCreate):
    session_id = len(sessions) + 1
    # ✏️ [실습 1] sessions 딕셔너리에 새 세션을 저장하세요.
    # 키는 session_id, 값은 id/user_id/bot_name/messages 필드를 가진 딕셔너리입니다.
    sessions[session_id] = {
        # 여기에 세션 정보를 채워보세요
    }
    return sessions[session_id]

@router.get("/{session_id}")
def get_session(session_id: int):
    # ✏️ [실습 2] session_id가 sessions에 없으면 404 에러를 반환하세요.
    # 여기에 조건문과 HTTPException을 작성하세요
    return sessions[session_id]

@router.delete("/{session_id}")
def delete_session(session_id: int):
    # ✏️ [실습 3] sessions에서 session_id를 삭제하세요.
    # pop(key, None)을 사용하면 키가 없어도 에러 없이 처리됩니다.
    # 여기에 삭제 코드를 작성하세요
    return {"status": "deleted", "session_id": session_id}
```

- 힌트
    
    - 세션 저장: `sessions[session_id] = {"id": session_id, "user_id": ..., "bot_name": ..., "messages": []}` 형태
    - 없는 세션 조회: `if session_id not in sessions:` 로 체크 후 `raise HTTPException(status_code=404, detail="...")`
    - 딕셔너리 삭제: `sessions.pop(session_id, None)` (두 번째 인자 None = 키 없어도 에러 없음)
- 정답
    
    ```python
    @router.post("/")
    def create_session(request: SessionCreate):
        session_id = len(sessions) + 1
        sessions[session_id] = {
            "id": session_id,
            "user_id": request.user_id,
            "bot_name": request.bot_name,
            "messages": [],
        }
        return sessions[session_id]
        # {"id": 1, "user_id": 1, "bot_name": "카카오봇", "messages": []}
    
    @router.get("/{session_id}")
    def get_session(session_id: int):
        if session_id not in sessions:
            raise HTTPException(status_code=404, detail="세션을 찾을 수 없습니다")
        return sessions[session_id]
    
    @router.delete("/{session_id}")
    def delete_session(session_id: int):
        sessions.pop(session_id, None)
        return {"status": "deleted", "session_id": session_id}
        # {"status": "deleted", "session_id": 1}
    ```

### Swagger UI란?

<aside> 📌

- 브라우저에서 직접 API를 호출해볼 수 있어요
- 팀원이나 프론트엔드 개발자와 **API 명세를 공유**할 때 유용해요 </aside>

### 명세 → 구현 → 검증 흐름 실습

<aside> 🔄

**흐름**

1. **명세**: 어떤 API가 필요한지 먼저 정의
2. **구현**: FastAPI 코드로 작성
3. **검증**: Swagger UI에서 직접 호출해서 확인 </aside>

**Step 1 — 명세 정의**

<aside> 📋

`POST /ai/analyze`

- 설명: 텍스트를 분석해서 감정을 반환
- 요청: `{"text": "오늘 너무 행복하다"}`
- 응답: `{"sentiment": "positive", "confidence": 0.95}` </aside>

**Step 2 — 구현**

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="카카오 AI API", version="0.1.0")

class AnalyzeRequest(BaseModel):
    text: str

class AnalyzeResponse(BaseModel):
    sentiment: str
    confidence: float

@app.post("/ai/analyze", response_model=AnalyzeResponse)
def analyze_sentiment(request: AnalyzeRequest):
    positive_words = ["행복", "좋아", "최고", "감사"]
    is_positive = any(word in request.text for word in positive_words)

    return AnalyzeResponse(
        sentiment="positive" if is_positive else "neutral",
        confidence=0.95 if is_positive else 0.6,
    )
    # POST {"text": "오늘 너무 행복하다"}
    # → {"sentiment": "positive", "confidence": 0.95}
```
