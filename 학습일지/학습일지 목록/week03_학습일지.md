## 📚 이번 주 학습한 것

### 1. DOM & 비동기 및 React 입문
- **핵심 개념**: DOM 제어, 비동기 데이터 처리(async/await, fetch), React 컴포넌트, JSX, `useState` 및 `useEffect`
- **내가 이해한 방식 (왜 그렇게 동작하는가)**:
  - 바닐라 JS로 DOM을 직접 조작할 때는 브라우저가 화면을 그리는 비용(Reflow/Repaint)이 자주 발생해 비효율적이다. 반면 React는 상태가 변경되면 가상 DOM(Virtual DOM)을 활용해 이전 트리와 비교(Reconciliation)한 뒤 변경된 '최소한의 차이'만 실제 DOM에 반영하여 렌더링 성능을 최적화한다는 작동 방식을 배웠다.
  - `useEffect`는 UI 렌더링이라는 컴포넌트 고유의 순수 역할 외에, API 호출이나 외부 리스너 등록 같은 '부작용(Side Effect)'을 렌더링 흐름 밖으로 분리하기 위해 사용한다. 특히 이펙트 내부에서 등록한 리스너가 컴포넌트 소멸 후에도 메모리에 잔존해 오작동을 일으키는 메모리 누수(Memory Leak)를 방지하려면, `cleanup` 함수를 통해 리스너를 정리해 주는 해제 단계가 필수적임을 이해했다.

### 2. React 구조와 상태 관리
- **핵심 개념**: 컴포넌트 분리 전략(SRP), Props(단방향 데이터 흐름), Lifting State Up(상태 끌어올리기), Custom Hook
- **내가 이해한 방식 (왜 그렇게 동작하는가)**:
  - React가 단방향 데이터 흐름(부모 -> 자식)을 고수하는 이유는 데이터가 변경되었을 때 어디서부터 발원했는지 상태의 변화 궤적을 역추적하기 쉽게 하기 위함이다. 양방향 바인딩 구조라면 형제 컴포넌트끼리 직접 상태를 수정하여 데이터 스파게티가 될 수 있지만, React는 이를 막기 위해 공통 부모로 상태를 끌어올리는 `Lifting State Up` 패턴을 강제한다.
  * 복잡한 비동기 로직이나 상태 관리 코드가 컴포넌트 내에 혼재되면 가독성이 떨어지고 재사용이 어렵다. 이를 React API를 포함하는 독립된 자바스크립트 함수인 `Custom Hook`으로 분리함으로써 UI를 그리는 역할과 비즈니스 로직을 명확하게 격리(관심사 분리)할 수 있다.

### 3. Tailwind CSS 스타일링
- **핵심 개념**: Utility-First CSS, 반응형 설계(sm, md, lg), hover/focus 상태 제어, Flexbox 레이아웃
- **내가 이해한 방식 (왜 그렇게 동작하는가)**:
  - Tailwind는 스타일 마크다운처럼 클래스 하나하나가 명확한 단일 스타일을 지닌 Utility-First 원칙을 제공한다. 이는 별도의 CSS 파일을 생성하거나 의미 없는 클래스 작명(`.submit-button-container-v2` 등) 고민을 없애주어 순수 화면 구현 생산성을 대폭 늘린다.
  - 미디어 쿼리를 일일이 작성할 필요 없이 `md:flex-row`처럼 모바일 우선(Mobile-First) 접두사를 사용해 점진적으로 화면 스케일을 확장하는 반응형 설계를 구현할 수 있다.

## 🧱 막혔던 지점 & 해결 과정
- **문제 상황**: 이번 주차 학습 도중 엘리스(Elice) 학습 사이트 서버가 계속 불안정하여 동영상 강의가 중간에 뚝뚝 끊기거나 온라인 실습 컨테이너 서버 연결이 강제 종료되는 등의 장애를 겪었다. 이로 인해 마감 요건을 맞추기 위한 학습 흐름이 완전히 끊겼고 코드 실습이 올바르게 진행되지 않는 병목 현상이 발생했다.
- **시도한 방법**: 서버 복구만을 마냥 기다리면 월요일 1시 제출 기한을 맞출 수 없다고 판단해, 온라인 컨테이너를 우회하여 로컬 PC에 Vite를 활용한 독립된 React 개발 환경을 수동으로 급히 구축했다. 강의 문서에 수록된 컴포넌트 분리 및 상태 끌어올리기 실습 예제를 로컬 프로젝트로 복사해와 `npm run dev`로 확인하며 실습을 로컬 포트에서 지속했다.
- **최종 해결 / 참고 자료**: 로컬에 환경을 세팅한 덕분에 온라인 실습 서버 장애와 무관하게 컴포넌트 분리와 Props 전달 코드를 무사히 작성하여 실질적인 실습 목적을 달성했다. 돌발적인 인프라 이슈 상황에서도 대안을 찾아 로컬 환경 세팅을 통해 학습의 맥락을 끊지 않고 주차 과제를 안전하게 완수할 수 있었다.
