## ⚡비동기 프로그래밍 (async/await)

<aside> 📌

**핵심 개념**

- **동기**: 한 작업이 끝날 때까지 기다린 뒤 다음 작업 시작
- **비동기**: 기다리는 동안 이벤트 루프가 다른 작업을 번갈아 처리
    - 라면 끓일 때 물 끓는 동안 봉지 뜯고 파 써는 것처럼, 대기 시간을 낭비하지 않는 방식(이벤트 루프가 순서를 조율함)
- FastAPI는 **비동기 기반**이라 동시에 여러 요청이 와도 효율적으로 처리 </aside>

![image (3).png](attachment:36729e60-2d83-4d3f-a5c1-cc7bc5983083:image_\(3\).png)

![image (4).png](attachment:c80e4ff2-d7bf-4cb1-905e-b3a7ded13e47:image_\(4\).png)

```jsx
import asyncio

async def task_a():
	print("A: start")
	await asyncio.sleep(1)  # 여기서 제어권 양보
	print("A: end")

async def task_b():
	print("B: start")
	await asyncio.sleep(2)
	print("B: end")

async def main():
	await asyncio.gather(task_a(), task_b())

asyncio.run(main())

# A: start  [0.0초] task_a 시작. 첫 번째 프린트 출력 후 sleep(1)을 만나 제어권을 넘김
# B: start  [0.0초] task_a가 넘겨준 제어권을 받아 task_b 시작. sleep(2)를 만나 제어권 넘김
# -------- (약 1초 동안 두 태스크 모두 대기 상태 / 이벤트 루프 작동 중) --------
# A: end    [1.0초] 1초가 지나 일어난 task_a가 남은 코드 실행 후 완전히 종료
# -------- (다시 약 1초 동안 task_b의 남은 시간 대기) --------
# B: end    [2.0초] 총 2초가 지나 일어난 task_b가 마지막 코드 실행 후 완전히 종료
```

```python
import asyncio

async def fetch_ai_response(message: str) -> str:
    await asyncio.sleep(1)  # AI API 호출 대기 — 이 동안 이벤트 루프가 다른 요청을 처리
    return f"AI 답변: {message}에 대한 응답"
```

|**구분**|**동기 (Synchronous)**|**비동기 (Asynchronous)**|
|---|---|---|
|**개념**|앞의 작업이 끝나야 다음 작업을 시작|앞의 작업을 기다리지 않고 다음 작업을 시작|
|**장점**|코드가 직관적이고 이해하기 쉬움|대기 시간을 낭비하지 않아 **서버 성능과 처리량이 극대화됨**|
|**단점**|작업이 오래 걸리면 서버 전체가 멈춤(블로킹)|코드가 복잡해지고, 실행 순서를 예측하기 어려움|
|**언제 쓰나요?**|- 단순한 계산 로직||

- 일반적인 DB 조회 | - **외부 API 호출** (OpenAI, 카카오 API 등)
- 파일 다운로드/업로드
- 대규모 트래픽 처리 |

## 🧪 테스트 (pytest)

<aside> 📌

**핵심 개념**

- 코드가 의도한 대로 작동하는지 **자동으로 검증**하는 도구
- 함수 단위 테스트부터 FastAPI 엔드포인트 테스트까지 그대로 확장 가능
- `assert 실제값 == 기대값` — 달라지면 테스트 실패 알림 </aside>

#### 사전 준비 (가상환경 및 라이브러리 설치)

실습을 시작하기 전, 터미널에서 아래 명령어를 순서대로 실행해 주세요.

```bash
# 1단계: 가상환경 생성 및 활성화
python -m venv venv

# (Mac 사용자용 활성화)
source venv/bin/activate

# (Windows 사용자용 활성화 - CMD/PowerShell 공통)
.\\venv\\Scripts\\activate

# 2단계: 필수 라이브러리 및 테스팅 툴 설치
pip install fastapi uvicorn "uvicorn[standard]"
pip install pytest pytest-asyncio

# 3단계: 테스트 실행
pytest
```

- 파일명은 `test_`로 시작하거나, `_test`로 끝나야 합니다.

유효성 검사하는 `validate_message()` 함수를 검증하는 테스트 예시를 실행해봅시다.

```python
def validate_message(message: str) -> str:
    # 1. 메시지가 비어있을 때
    if not message:
        return "메시지를 입력하세요"
    
    # 2. 메시지가 500자를 초과할 때 (501자부터)
    if len(message) > 500:
        return "메시지가 너무 깁니다"
    
    # 3. 통과했을 때 (정상 메시지)
    return message
```

```python
def test_validate_message():
    assert validate_message("안녕") == "안녕"
    assert validate_message("") == "메시지를 입력하세요"
    assert validate_message("a" * 501) == "메시지가 너무 깁니다"
```

## 🌐 HTTP & API 기초

<aside> 📌

**핵심 개념**

- **HTTP**: 클라이언트(앱, 브라우저)와 서버가 데이터를 주고받는 약속된 규칙
- **API**: 앱끼리 데이터를 주고받는 창구
- **메서드**: `GET` (조회) · `POST` (생성) · `PUT` (전체 교체) · `PATCH` (부분 수정) · `DELETE` (삭제)
- **상태코드**: `200` OK · `201` Created · `400` 잘못된 요청 · `401` 인증 필요 · `404` 없음 · `500` 서버 오류 </aside>

![image.png](attachment:dcfd73ce-78ff-4e4c-89b3-f1b90dca23dc:f68c214d-15a6-41e4-8a16-c460454106eb.png)

HTTP 메서드와 상태코드를 배웠다면, 이제 **"URL(주소)을 어떻게 지어야 누구나 이해하기 쉬울까?"**를 고민해야 합니다. 전 세계 개발자들이 약속한 가장 유명한 설계 규칙이 바로 **RESTful API**입니다.

### 📐 API 주소를 짓는 글로벌 규칙: RESTful API

<aside> 🤔

**왜 규칙이 필요할까요?**

만약 개발자 마음대로 주소를 짓는다면 어떨까요?

- A 개발자: GET /getUserData
- B 개발자: POST /deleteUser (삭제인데 POST를 씀)
- C 개발자: GET /user/123/update

이렇게 중구난방이면 다른 개발자가 API를 사용할 때마다 "이 주소는 무슨 뜻이지?" 하고 매번 물어봐야 합니다.

</aside>

RESTful API의 핵심은 URL을 "동작"이 아닌 **자원(Resource) 중심**으로 설계한다는 점입니다. `/getUser` 대신 `/users/123`처럼 명사 형태로 자원을 표현하고, 무엇을 할지는 HTTP 메서드로 구분합니다.

```
GET    /users/123      → 123번 유저 정보 조회
POST   /chat/messages  → 새 메시지 전송
PUT    /users/123      → 123번 유저 정보 전체 교체
PATCH  /users/123      → 123번 유저 정보 일부 수정
DELETE /sessions/abc   → 세션 종료
```

## 🔗 오늘 배울 내용 미리보기

<aside> 🔗

**API를 개발할 때 겪을 수 있는 문제들**

1. **데이터 검증의 번거로움:** 클라이언트가 나이(int) 자리에 문자열("스무살")을 보내면 서버 에러가 발생합니다. 이를 막기 위해 모든 요청마다 타입 검사 코드를 작성해야 합니다.
2. **API 문서화의 부담:** 프론트엔드 개발자와 협업하려면 "어떤 주소로, 어떤 데이터를 보내면, 어떤 응답이 오는지"를 일일이 문서(엑셀, 위키 등)로 정리해서 공유해야 합니다.

오늘 배울 **FastAPI**와 관련 도구들은 이러한 실무의 비효율을 크게 줄여줍니다.

1. **Pydantic (BaseModel)**
    
    👉 API로 들어오는 데이터의 타입과 형태를 정의합니다. 규칙에 맞지 않는 데이터가 들어오면, 개발자가 별도의 코드를 짜지 않아도 자동으로 에러(422)를 반환하여 서버를 보호합니다.
    
2. **Swagger UI**
    
    👉 작성한 파이썬 코드를 기반으로, 브라우저에서 직접 테스트해 볼 수 있는 API 명세서가 **자동으로 생성**됩니다. 별도의 문서 작업 없이 이 화면만 공유하면 협업이 가능해집니다.
    

</aside>
