### HTML 소개와 기본 구조

- HTML
	- 웹 페이지와 그 내용을 구조화 하기 위한 마크업 언어
	- 웹 페이지의 구조와 내용을 정의
	- 태그를 이용하여 텍스트, 이미지, 링크 등의 요소를 표시하고 구성
	


- HyperText Markup Language
	- HyperText 웹사이트에서 링크를 클릭해 다른 문서나 사이트로 즉시 이동할 수 있는 기능
	- Markup 태그를 사용해 문서에서 각 부분의 의미를 표시하는것
	- Language
	- 웹에서 자유롭게 오갈 수 있는 웹 문서를 생성하는 마크업 언어.


- 태그
	- 요소의 시작과 끝을 표시하는 마크업 기호
	- 텍스트, 이미지가 있어야할 부분을 표시하는 용도
- 태그의 규칙
	- <>로 붂인 부분이 태그
	- 여는 태그와 닫는 태그는 이름이 같아야한다
	- / 유무로 구분한다

- <태그이름 속성 = "속성값">콘텐츠 </태그이름>
	- 태그이름 : 해당 태그가 어떤 콘텐츠를 표시하는지 의미함
	- 콘텐츠: 태그 사이에 태그가 실제로 표시할 콘텐츠(내용) 작성
	- 속성: 태그에 부가기능을 추가하기 위해 작성하는것으로 선택 사항
	- 속성값: 속성에 부여된 기능이나 설정을 구체적으로 지정하는값


- Self-colsing tags
	- 태그 내부에 콘텐츠나 텍스트를 포함하지않아 \</tag> 형태로 닫을 필요가없는 태그들
	- 자체적으로 닫아줘도 되지만, 브라우저는 동일하게 해석함
	- \<tag>와 \<tag/>는 브라우저에게 동일하게 해석됨
		- 이왕이면\<tag/> 형태도 닫는것을 추천함
	- \<br/> 줄바꿈
	- \<img/> 이미지
	- \<input/> 입력창

### HTML 요소와 텍스트

- HTML요소
	- HTML 문서나 웹 페이지를 이루는 개별적인 요소
	- 태그를 이용해 표현
	- 웹페이지의 내용을 의미적으로 주고화
	- 브라우저에 해당 내용을 어떻게 표시해야하는지 알려주는 역할

제목 태그(hx): 제목이나 부제목을 표현 h1~h6까지 존재하고 h1이 가장 큰 주제 표현

단락 태그(p): 문단을 표현 문단 내용을 열린 태그와 닫힌 태그 사이에 입력

순서가 없는 목록과 목록 요소 태그 (ul: unordered list, ol: ordered list, li: list item): 순서가 정해지지 않은 목록과 그 목록의 각 요소들 표현. 


- 컨테이너 태그
	- 요소들을 묶어서 구조화 하거나 스타일링을 적용하는 역할을 하는요소
	- 웹페이지의 레이아웃과 내용을 조직화
	- 아무기능이없지만, 구조화에 탁월해 많이 사용한다
- \<div>: Block 수준의 컨테이너 요소 - 한 줄을 모두 차지, 다른 요소가 개입 못함(Division: 분할, 구역)
	- 새로운 줄 작(블록 수준의 요소는 항상 새로운 줄에서 시작)
	- 전체 너비 차지(블록 수준의 요소는 부모 요소의 너비를 모두 차지)
	- 최대한 적게 사용해보자


-  \<span>: Inline 수준의 컨테이너 요소 - 한줄에 여러개가 들어감, 다른 요소 개입 가능.(Span: 폭, 범위, 걸치다)
	- 줄 내에서 표시
	- 텍스트 단락 안에서 줄바꿈 없이 일부 텍스트만 묶어 스타일 적용을 위해 사용

태그 안에 다른 태그가 자유롭게 들어갈 수 있다.

### CLASS와 ID

- 전역 속성
	- 모든 HTML요소에서 사용할 수 있는 콩통 속성을 의미
	- 태그의 속성 부분에 들어가는 속성
	- class: 페이지 내에서 여러 요소에 동일한 스타일을 적용하고 싶을 때 같은 class이름을 각각의 요소에 부여
		- HTMl 요소에 하나 이상의 클래스를 할당하는데 사용
		- 요소를 그룹화, 식별하기위해 활용
		- CSS, JavaScript에서 클래스 이름을 활용하여 요소 선택
		- 요소에 class 속성을 추가하고 속성 값으로 클래스 이름지정
		- 한 요소에 여러 클래스 할당 가능
		- 여러 요소에 여러 클래스 이름 반복적으로 할당 가능
		- 클래스는 공백으로 구분(class = "main important")
		- ```
		  <body>
			  <p class = "main">
				  이 부분은 주요 내용이 담긴 부분입니다.
			  </p>
		  </body>
		  ```
	- id: 요소에 고유한 식별자를 부여
		- HTML 요소에 고유한 이름을 부여해 요소 식별에 사용
		- 해당 HTML 문서 내에서 유일해야한다
		- 하나의 요소에 한개의 id만 할당 가능하다
		- 요소에 id 속성을 추가하고 속성값으로 id이름 지정
		- CSS, JavaScript에서 id를 활용하여 요소 선택
		- ```
		  <body>
			<p id = "content">
				이 부분은 주요 내용이 담긴 부분입니다.
			</p>
		  </body>
		  ```
	- id, class 이름 규칙
		- 대소문자 구분
		- 공백이나 특수 문자 포함 불가
		- 숫자나 문자로 시작, 숫자로만 이루어진 경우 사용 불가


| 구분      | class                                   | id                                |
| :------ | :-------------------------------------- | :-------------------------------- |
| **공통점** | 요소에 대한 속성으로, 요소를 식별하거나 선택자로 사용 가능       | 요소에 대한 속성으로, 요소를 식별하거나 선택자로 사용 가능 |
| **차이점** | 한 페이지 내에서 동일한 class 이름을 여러 요소에 중복 사용 가능 | 한 페이지 내에서 고유한 id 명은 한 번만 사용 가능    |
|         | 하나의 요소에 여러 class 사용 가능                  | 하나의 요소에는 오직 하나의 id만 사용 가능         |
|         | 여러 요소를 동시에 선택할 때 선택자로 사용                | 페이지 내에서 특정 요소 하나만 선택할 때 선택자로 사용   |

### 이미지와 링크 요소


- 이미지 요소
	- HTML에서 이미지를 표시하기 위해 사용함.
	- \<img> 태그 이용
		\<img src = "이미지_파일_상대_경로" alt = "대체+텍스트" />
	- src 속성: 이미지의 파일 경로 지정(필수)
	- alt 속성: 이미지를 설명하는 대체 텍스트(이미리가 로드되지 않았을때 표시되는 문구, 시각 장애인을 위해 중요한 역할)

- 링크 요소
	- 파이퍼링크를 생성
	- \<a> 태그 이용: 다른 웹 페이지, 현재 페이지 내의 다른 위치로 이동 가능
	- \<a href = "링크_URL" target = "_blank">링크_텍스트</a>
	- href 속성: 링크의 대상 URL을 지정(필수)
	- target 속성
		- \_self: 현재 탭에서 이동(default)
		- \_blank: 새로운 탭에서 이동

### 테이블 요소

- 테이블 생성
	- \<table>태그 이용
	- 데이터를 행(row)과 열(column)의 조합으로 구조화 하여 표현
		- \<tr> 행을 나타내는 요소(자손으로 th, td 태그를 포함)
		- \<th> 헤더 셀을 나타냄(열 또는 행의 제목 표시), 굵은 글씨로 표시
		- \<td> 데이터 셀을 나타냄(실제 데이터 표시)

### 폼 요소

- 폼 양식
	- 사용자로부터 입력을 받을 수 있는 HTML양식
	- \<form> 폼 요소들을 감싸며 HTML의 form양식을 정의
	- \<form> 태그의 속성
		- method: 사용자가 입력한 내용들을 서버쪽으로 어떻게 넘겨줄지 지정. GET, POST 방식이 존재
		- name: 폼의 이름을 지정 한 문서 내에 여러개의 form tag가 존재할 경우, 폼들을 구분하기 위해 사용
		- action: 데이터를 전송할 URL이나 서버 쪽의 script 파일명, 보통 서버의 프로그램과 연결되는 주소
	- \<form>에 들어가는 태그들
		- \<lable> 태그
			- \<input>, \<textarea>와 같은 엽력 요소와 연결된 라벨요소
				- label 요소를 클릭하면 입력 요소가 활성화 됨
			- for 속성: 연결할 입력 요소의 id 값
			- \<label for = "password"> 비밀번호: \<\label>
		- \<input> 태그
			- 가장 기본적인 입력 요쇼, self-closing tag
			- type 속성: input 유형을 결정하는 속성
				- input 요소에는 여러 type이 존재하며, 각 type마다 사용할 수 있는 속성이 다름
				- | 속성값 | 설명 |
| :--- | :--- |
| text | 한 줄 길이의 텍스트를 입력할 수 있는 필드 |
| email | 이메일을 입력할 수 있는 필드 |
| url | url 형식을 입력할 수 있는 필드 |
| password | 비밀번호를 입력할 수 있는 필드 |
| radio | 주어진 여러 항목에서 1개만 선택할 수 있는 라디오 버튼을 생성하는 필드 |
| checkbox | 주어진 여러 항목에서 2개 이상 선택할 수 있는 체크박스를 생성하는 필드 |
| button | 일반적인 버튼을 생성(JavaScript를 이용해 효과 생성) |
| submit | 입력된 값들을 서버에 제출하는 버튼을 생성 |
			- placeholder 속성: 입력 내용에 대한 힌트 제공
			- name 속성: form 제출 시 해당 입력 값의 이름(key)이 되는 값
			- 이 외에도 input 요소에는 다양한 속성이 존재(ex. value, name, disabled)

### Semantic HTML5 요소

- Semantic 태그(Semantic: 의미의, 의미론적인, 의미가 통하는 이라는뜻의 형용사)
	- \<div> 태그와 동일한 컨테이너 태그
	- 단순히 구획만 나누는 \<div> 와는 다르게, 해당 요소의 역할과 의미를 예측 가능하게 함
	- 태그만 보고도 역할이나 위치 파악 가능
- Semantic 태그의 출현
	- HTML5 이전에서는 div태그만 사용. 이름 지정시 제약이 없어 이후에 문서 구조 파악이 어려움
	- Semantic 태그 이용시에 HTML 문서의 가독성과 유지보수성이 좋아짐
	- 웹 브라우저가 HTML 소스 코드만 보고도 어느 부부이 제목인지, 본문인지 쉽게 알 수 있음
		- 글을 소리로 읽어주는 기계(스크린 리더기)를 사용하는 경우 사이트의 구조를 소리로 구분 가능하게 함
	- 검색 엔진이 검색 수행시 HTML 태그를 분석 할 수 있음(\*SEO)

| **태그명**               | **의미**                                                                |
| --------------------- | --------------------------------------------------------------------- |
| `<header></header>`   | 웹 페이지, 섹션의 헤더 (제목, 로고 등 포함)                                           |
| `<nav></nav>`         | 네비게이션 링크 (웹 페이지의 주요 링크 포함)                                            |
| `<main></main>`       | 주요 콘텐츠 영역                                                             |
| `<article></article>` | 독립적인 콘텐츠 블록<br><br>  <br><br>(뉴스 기사, 블로그 글, 포스트처럼 독립적으로 배포 가능한 내용 처리) |
| `<aside></aside>`     | 사이드바 표시                                                               |
| `<footer></footer>`   | 웹 페이지, 섹션의 footer(저작권 정보, 연락처 정보, 관련 링크)                              |

### CSS 소개 및 기본 문법

- CSS
	- 웹페이지의 스타일링, 레이아웃 등의 디자인을 지정하는 언어
	- 디자인과 스타일링: CSS를 사용해 웹 페이지의 레이아웃, 색상, 글꼴, 크기 등을 지정해 웹사이트를 디자인
	- 일관성과 유지 보수성: CSS를 사용하면 스타일을 별도의 파일에 저으이해 여러 페이지에 일관된 스타일을 적용가능하고, 스타일 별경이 필요할 때 스타일 시트만 수정해 일관성 유지 가능
	- 접근성: CSS를 사용해 웹 페이지의 구조와 스타일을 분리해 웹 접근성 향상
	- CSS(Cascading Style Sheets)
		- Cascading: 폭포같은, 계속되는, 연속적인 형용사
		- 상위 요소의 스타일이 하위 요소에 상송되고, 동일 요소에 여러 스타일 규칙이 있으면 우선순위에 따라 적용
	- CSS는 웹 페이지의 다양한 요소에 스타일을 적용하는 규칙의 집합을 위에서 아래로 흐르듯이 우선순위에 따라 순차적으로 적용하는 시스템을 의미

- CSS 구문과 기본문법
	- 선택자(Selector): 스타일을 적용할 HTML요소를 선택하는 부분
		- 원하는 요소를 선택하기 위해 태그이름, 글래스, 아이드 등을 사용
		- 스타일을 적용할 대상을 지정하는 역할
	- 선언(Declaration): 선택한 HTML 요소에 적용할 스타일을 정의하는 부분
		- 속성(Property): 값(Values)의 형식으로 적용
		- 속성은 스타일을 어떻게 변경할지를 나타냄
		- 값은 해당 속성에 적용할 구체적인 값
	- ```
	  /*
	  선택자(태그이름)와 선언(속성과 값)
	  p요소에 글자의 색상을 파랑색, 크기를 16px로 적용
	  */
	  p{
		  color: blue;
		  font-size: 16px;
	  }
	  ```

- HTML 파일에 CSS스타일을 적용하는방법
	- 외부 스타일 시트
		- CSS코드를 별도의 파일로 저장하고, HTML 문서에서 외부 스타일 시트 파일을 참조하여 스타일을 적용
		- 외부 스타일 시트 파일은 .css확장자를 가지고, \<link>태그를 사용하여 HTML 문서와 연결
		- 외부 스타일 시트를 사용하는것이 가장 권장되는 방법
			- 재사용성과 유지 보수성 측면에서 좋은 성능을 보여준다.
	- 내부 스타일 시트
		- \<style>태그를 사용하여 HTML 문서 내부에 CSS스타일을 정의(\<style>태그는 \</hea>태그 내부에 위치)
		- 이 방식 이용시 해당 HTML 문서에서만 스타일 적용
	- 인라인 스타일
		- HTMl 태그의 style 속성을 사용하여 해당 태그에 직접 스타일 지정
		- 한번 설정된 스타일을 변경하기가 매우 어렵고, 스타일 시트를 이용하는 많은 이점을 잃게 되기 때문에 꼭 필요한 경우에만 사용해야함

스타일 적용의 우선순위
1. 인라인 스타일(HTML 요소 내부에 위치)
2. 내부.외부 스타일 시트(HTML 문서의 head 요소 내부에 위치)
3. 웹 브라우저 기본 스타일(user agent style)
- 인라인 스타일이 적용된 태그는 스타일 시트에 상관없이 무조건 인라인 스타일 적용
- 내부, 외부 스타일 시트는 가장 마지막에 적용된 스타일 시트가 적영

### 기본 선택자와 복합 선택자

- 선택자
	- CSS에서 스타일을 적용할 대상을 선택하는데에 사용
	- 단순 선택자
		- 전체 선택자
			- 모든 HTML 요소에 스타일 적용시에 사용
			  - \*(별표)를 사용한다
			  - 초기화 스타일 시트(reset stylesheet)을 만들거나 기본 스타일 설정시에 사용한다
			  - 모든 요소에 일괄적으로 스타일을 적용하는것이 문제가 될 수 있으므로 사용에 주의할것
		 - 태그 선택자
			 - HTML의 태그(p, h1, div 등)으로 요소를 선택해 스타일을 지정하는 선택자
			 - 특정 태그가 사용된 HTML 요소에 모두 동일한 스타일 적용
		 - 클래스 선택자
			 - '.'을 이용하여 클래스 이름으로 스타일을 지정하는 선택자
			 - 동일한 클래스(같은 클래스이름)을 가진 여러 요소에 같은 스타일 적용
			 - 같은 태그를 사용했어도 특정 요소에만 다른 스타일을 적용하고 싶을때에도 사용
		 - ID선택자
			 - '#'을 이용하여 ID 이름으로 스타일을 지정
			 - 하나의 요소에 대해 고유한 스타일 지정시 사용
			 - 한 문서 내에서 한번만 적용가능
	 - 복합 선택자: 2개 이상의 단순 선택자를 조합해 특정 요소 선택 가능
		 - 자손 선택자
			 - 공백을 사용하여 하위 요소를 선택
			 - 부모 요소와 하위 요소의 관계를 정확히 지정하여 스타일을 적용하ㅏㄴ다
		 - 자식 선택자
			 - '>'를 사용하여 직접적인 자식 요소를 선택
			 - 바로 아래의 자식 요소만 스타일이 적용
		 - 그룹 선택자
			 - 쉼표(',')를 사용하여 여러개의 선택자를 그룹으로 묶어 스타일 지정
			 - 같은 스타일을 여러 요소에 적용할 때 편이
			 - 왼쪽 예시의 경우 제목 1, 제목 2, 문단 1 모두 초록색으로 표시되고 가운데 정렬
	- 자손, 자식, 부모.
		- ```<body>
    <div>
        <h1>
            제목
        </h1>
        <p>
            문단
            <span>
                p의 자식
            </span>
        </p>
    </div>
</body>
			  
		  ```
		- h1은 div의 자식이자 자손
		- psms div의 자식, 자손이면서 span의 부모
		- span은 p의 자식이자 div의 자손

### 가상 선택자와 스타일 규칙

- 가상선택자
	  선택자에 추가하는 키워드로, 특정한 목적을 위해 사용되는 선택자
	  요소의 내부 구조나 상태에 따라 스타일을 다르게 적용하고자 할 때 유용
	  HTML 문서의 수정 없이 CSS만으로 디자인적 요소를 추가할 수 있음
	- 가상 클래스 선택자: 특정 상태에 있는 요소를 선택하여 스타일 적용
		- :hover
		  마우스 커서가 요소 위에 있을 때 스타일이 적용
		- :active
		  사용자가 요소를 활성화(클릭 등) 했을 때 적용
		- :focus
		  요소가 포커스를 받았을 때(입력 필드에 커서가 들어간 경우 등)
	- 가상 요소 선택자: 특정 요소의 일부분에 스타일 적용
		- ::before
		  요소의 앞에 컨텐츠(지정 내용(텍스트나 이미지) 추가)
		- ::after
		  요소의 앞에 컨텐츠(지정 내용(텍스트나 이미지) 추가)
		- ::first-letter
		  요소의 첫 글자에 적용

- 선택자들의 우선순위
	- 우선순위이가 높은 스타일이 먼저 적용된다.
		- !important > 인라인 선언 방식 > id 선택자 > class 선택자 > 태그 선택자 > 전체 선택자 *
	- 우선순위가 같을 경우 선택자의 개수가 많은 CSS가 우선 적용된다.
	- 순위가 미확정이라면, 가장 뒤 쪽에 작성된 스타일이 적용
	- 개수가 아무리 많아도 우선순위 높은 스타일이 우선 적용

### 텍스트 스타일링

- 텍스트 스타일링: 폰트

### 박스 모델

- 박스모델: HTML 요소들은 모두 사각형의 박스로 취급한다. 각 박스의 크기를 결정하는 개념이다.
	- Box-sizing: width, height 대상 영역 변경 가능. 크기 계산 기준 변경(속성명 Box_sizing)
			- Content-box: width, height 프로퍼티 값이 content 영역을 의미(기본 값)
			- Border-box : width, height 프로퍼티 값이 content 영역, padding, border가 포함된 값을 의미
	- Content: 요소의 텍스트나 이미지 등의 실제 내용이 위치하는 영역(속성명 width, height)
	
	- Padding: 테두리 안쪽에 위치하는 요소의 내부 영역(속성명 padding)
	- Margin: 테두리(border) 바깥에 위치하는 요소의 외부 여백 영역(속성명 Margin)
		- - 마진 상쇄
	- 어떤 두개 이상의 블록 요소의 우 아래 마진이 겹치면, 제일 큰 크기의 단일 마진으로 결합하는 현상
	- 인접해있는 Block 요소 사이에서만 발생함
	- 상하단의 마진에 관해서만 발생(좌우로는 발생하지않음)
		- 지정방식(padding 도 동일)
		- ||**margin: 25px 50px 75px 100px**|**margin: 25px 50px 75px**|**margin: 25px 50px**|**margin: 25px**|
|---|---|---|---|---|
|**margin-top**|25px|25px|25px|25px|
|**margin-right**|50px|50px|50px|25px|
|**margin-bottom**|75px|75px|25px|25px|
|**margin-left**|100px|50px|50px|25px|
	- Border: 테두리 영역으로 border 프로퍼티 값은 테두리의 두께를 의미한다(속성명(Border))
		- 가지는 값들
			- line-widht: border의 굵기
			- line-style: border 스타일
				- soild: 실선
				- dashed: 직사각형 점선
				- dottde: 원형 점선
			- color: border 색상








- Block 요소
	- width: 부모 컨테이너의 전체 너비
	- height: 내무 컨텐츠에 의해 결정
	- 항상 새로운 줄에서 시작(세로 방향으로 쌓임)
	- **상하좌우 marhin과 padding 적용 가능**
- Inline 요소
	- width와 height가 내부 컨텐츠에 의해 자동으로 결정
	- 줄바꿈 하지 않음(가로 방향으로 쌓임)
	- **padding은 상하좌우 모두 적용되지만 margin은 상하 적용이 되지않음(좌우는 적용됨)**

### 배경과 그라디언트

- 배경 색상 설정 방법
	- CSS의 background-color 속성을 사용하여 요소의 배경 색상 지정 가능
	- 색상이름, HEX 코드,  RGB값을 사용하여 색상 지정 가능
		- 색 이름: 미리 정의된 색상이름 사용
		- HEX 코드: 16지ㄴ수 색상 코드 사용
		- RGB값: RGB 색상 모델 사용
		- RGBA값: RGB와 더불어 알파채널 사용
			- A는 투명도
			- 0.0(완전 투명) ~1.0(완전 불투명)
```
/*배경색을 파란색으로 설정한 예시 */
.div-background{
background-color: blue;
}
```

- 배경 이미지 설정
	- background-image 속성을 사용하여 요소의 배경 이미지 설정 가능
	- 이미지 파일 경로를 지정하여 배경으로 사용할 이미지 설정
	- ```
	  /*배경 이미지를 설정한 예시 */
	  .div-background-image{
	  background-image: url('images/background.jpg');
	  }
	  ```

- 그라이던트를 활용한 배경효과
	- 두개 이상의 색상을 서서히 혹은 일정 간격으로 변화시키는 배경효과
	- 선형 그라디언트
		- ```
		  /*선형 그라디언트를 적용한 예시*/
		  .div-linear-gradiend{
		  background: linear-gradient(to right, red, blue);
		  }
		  ```
	- 원형 그라디언트
	- ```
	  /*원형 그라디언트를 적용한 예시*/
	  .div-radial-gradient{
	  background: radial-gradient(circle, yellow, green);
	  }
	  
	```

### Flex Layout

- Flex
	- 1차원 레이아웃을 위한 도구, 주 축과 교차축을 이용해 더 유연한 레이아웃 생성
	- container(부모), item(자식) 2가지 개념으로 구성
- display:flex; 를 **부모에게** 설정 후 flex의 다양한 속성을 이용해 자식 요소 배치
  flex container에 적용하는 속성
 -  |**속성**|**설명**|
|---|---|
|`display`|flex container를 정의|
|`flex-direction`|flex items의 주 축(main-axis) 설정|
|`justify-content`|주 축(main-axis)의 정렬 방법을 설정|
|`align-items`|교차 축(cross-axis)의 정렬 방법을 설정|
|`flex-wrap`|flex items의 여러 줄 묶음(줄 바꿈) 설정|
|`flex-flow`|`flex-direction`과 `flex-wrap`의 단축 속성|
	- display 속성	
	- |**속성과 값**|**의미**|
|---|---|
|`display: flex`|Block 특성의 flex container 정의|
|`display: inline-flex`|Inline 특성의 flex container 정의|
	- flex-direction 속성
	- |**속성과 값**|**의미**|
	|---|---|
	|`flex-direction: row` (기본값)|Items를 수평축(왼쪽에서 오른쪽)으로 표시 (주 축이 오른쪽 방향)|
	|`flex-direction: row-reverse`|Items를 row의 반대로 표시 (주 축이 왼쪽 방향)|
	|`flex-direction: column`|Items를 수직축(위에서 아래)으로 표시 (주 축이 아래쪽 방향)|
	|`flex-direction: column-reverse`|Items를 column의 반대 축으로 표시 (주 축이 위쪽 방향)|
	- justify-content 속성
	- |justify-content 값|정렬 방식 설명|
	|---|---|
	|`flex-start`|주 축의 시작점(왼쪽)으로 아이템들을 뭉쳐서 정렬|
	|`center`|주 축의 가운데로 아이템들을 뭉쳐서 정렬|
	|`flex-end`|주 축의 끝점(오른쪽)으로 아이템들을 뭉쳐서 정렬|
	|`space-between`|첫 아이템과 끝 아이템은 양 끝에 딱 붙고, 나머지 빈 공간을 동일하게 배분|
	|`space-around`|각 아이템의 좌우에 동일한 크기의 여백을 줌 (결과적으로 아이템 사이의 간격이 가장자리 간격보다 2배 넓어짐)|
	|`space-evenly`|아이템 사이의 간격과 양 끝 가장자리 간격을 모두 완벽하게 동일한 크기로 배분|
	- align-items 속성
	- |align-items 값|정렬 방식 설명|
	|---|---|
	|`flex-start`|교차 축의 시작점(위쪽)에 아이템들을 맞춰서 정렬|
	|`flex-end`|교차 축의 끝점(아래쪽)에 아이템들을 맞춰서 정렬|
	|`center`|교차 축의 가운데에 아이템들을 맞춰서 정렬|
	- flex-wrap 속성
	- |속성과 값|의미|
	|---|---|
	|`flex-wrap: nowrap` (기본값)|모든 Items를 여러 줄로 묶지 않음(1줄에 표시)|
	|`flex-wrap: wrap`|Items를 여러 줄로 묶음|
	|`flex-wrap: wrap-reverse`|Items를 wrap의 역방향으로 여러 줄로 묶음|
	- flex-flow 속성
	  flex_direction 과 flex-wrap을 한번에 지정하는 속성
	- flex item에 적용하는 속성
	- |속성|설명|기본값|속성값|설명|
	|---|---|---|---|---|
	|`order`|flex item의 순서를 결정|0|숫자|숫자 작을수록 앞 순서|
	|`flex-grow`|flex item의 증가 너비 비율을 설정|0|숫자|증가 비율|
	|`flex-shrink`|flex item의 감소 너비 비율을 설정|1|숫자|감소 비율|
	|`flex-basis`|flex item의 (공간 배분 전) 기본 너비 설정|auto|단위|px, em, rem, ...|
	|`flex`|`flex-grow`, `flex-shrink`, `flex-basis` 단축 속성||||
	|`align-self`|교차 축(cross-axis)에서 item의 정렬 방법을 설정|stretch|flex-start, flex-end, center|아이템 정렬|

### Grid Layout

- Grid
	- 2차원 레이아웃을 위한 도구
	- 행과 열을 이용해 복잡한 레이아웃 구현 

Grid Layout은 아이템을 넣기전에, 미리 공간을 만들고 작업한다.

- Grid Layout 구성요소
	- Grid container: 전체 Grid Layout을 감싸는 역할 수행
	- 사용하는 개념들
		- Grid track: Grid Layout에 존재하는 행 또는 열 의미
		- Grid line: Grid track을 구분하는 선을 의미
		- Grid cell: Grid Layout에서 가장 작은 단위요소. 테이블의 셀과 유사
		- Grid area: 다수의 Grid cell로 이루어진 영역

	- grid 박스 모델: grid 형태 정의
	
	- | **속성**                | **의미**                                           |
	| --------------------- | ------------------------------------------------ |
	| grid-template-rows    | row의 배치 결정                                       |
	| grid-template-columns | column의 배치 결정                                    |
	| grid-auto-columns     | grid-template-columns의 통제를 벗어난 위치에 있는 트랙의 크기를 지정 |
	| grid-auto-rows        | grid-template-rows의 통제를 벗어난 위치에 있는 트랙의 크기를 지정    |
	- grid cell의 간격 설정
	- | **속성**     | **의미**                       |
	| ---------- | ---------------------------- |
	| row-gap    | grid cell 사이의 row 간격을 설정함    |
	| column-gap | grid cell 사이의 column 간격을 설정함 |
	| gap        | grid cell 사이의 간격을 설정함        |

	- grid 형태 자동으로 정의
	- | **값**               | **의미**                                |
| ------------------- | ------------------------------------- |
| repeat(반복횟수, 반복값)   | 반복되는 값 자동으로 처리하는 함수                   |
| minmax(최소값, 최대값)    | 최소, 최대한의 폭 지정하는 함수                    |
| auto-fill, auto-fit | column의 개수를 미리 정하지 않고 설정된 너비만큼의 셀을 채움 |


- Grid item: Grid container에 속해 있는 하위 DOM 요소
	- Grid item의 속성

| **속성**                                              | **의미**                                                             |
| --------------------------------------------------- | ------------------------------------------------------------------ |
| `grid-column-start`                                 | 아이템이 시작하는 열(세로줄)의 라인 번호를 지정                                        |
| `grid-column-end`                                   | 아이템이 끝나는 열(세로줄)의 라인 번호를 지정                                         |
| `grid-column`                                       | `grid-column-start`와 `grid-column-end`를 한 번에 쓰는 단축 속성 (예: `1 / 3`) |
| `grid-row-start`                                    | 아이템이 시작하는 행(가로줄)의 라인 번호를 지정                                        |
| `grid-row-end`                                      | 아이템이 끝나는 행(가로줄)의 라인 번호를 지정                                         |
| `grid-row`                                          | `grid-row-start`와 `grid-row-end`를 한 번에 쓰는 단축 속성 (예: `1 / 2`)       |
| `gird-templates-areas`                              | 각 영역(grid area)에 이름을 붙이고, 그 이름을 이용해서 배치하는 방법                       |
| `grid-auto-flow`                                    | 아이템 자동 배치되도록 결정                                                    |
| `order`                                             | 각 아이템들의 시각적 나열 순서 지정                                               |
| `align-items`, `justify-items`, `place-items`       | 아이템들 세로, 가로, 세로+가로 정렬                                              |
| `align-content`, `justify-content`, `place-content` | 아이템 그룹 세로, 가로, 세로+가로 정렬                                            |
| `align-self`, `justify-self`, `place-self`          | 아이템 세로, 가로, 세로+가로 정렬                                               |

### Position

- 레이아웃
	웹 페이지 디자인의 핵심
	- 웹 페이지의 첫인상 결정
	- 사용자 경험(UX)과 직접 연결
	- 디자인의 목적과 내용에 따른 레이아웃 선택의 중요성
	- Dispaly 속성, Position 등을 이용해 구현 가능


- Display 속성
	- **None** : 화면에서 사라지고 공간을 차지하지도 않음
	- **Block** : 새로운 줄에서 시작, 가능한 모든 가로 공간 차지
	- **Inline** : 새 줄을 시작하지 않고, 필요한 만큼의 공간만 차지
	- **Inline-block** : inline처럼 새 줄을 시작하지 않지만, block처럼 높이와 너비 지정 가능
	- + **Flex**, **Grid**

- Position 속성
	HTML 문서 상에서 요소가 배치되는 방식 결정
	- **Relative** : positioning의 기준이 되는 요소로 지정
	- **Absolute** : 가장 가까운 relative 조상 요소에 대해 상대적으로 이동
	- **Fixed** : 뷰포트에 대해 상대적으로 이동
	- **Sticky** : 스크롤 위치에 따라 상대적, 절대적으로 동작
	- **Position**을 사용하면 **top/right/bottom/left** 속성으로 위치 조정 가능
