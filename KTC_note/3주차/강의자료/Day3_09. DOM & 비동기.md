### 1️⃣ DOM이란?

**DOM(Document Object Model)**은 HTML 문서를 구조화된 객체 형태로 표현한 것입니다. 브라우저가 HTML 파일을 읽으면, 각 태그를 **node 객체**로 변환하여 트리 구조를 만듭니다.

![DOM 구조](image%20(1).png.png)

* **주요 DOM 노드 유형**

| node 유형 | 설명 |
| --- | --- |
| **document** | 웹 페이지 전체. DOM 접근의 출발점 |
| **element** | 개별 HTML 요소 (`<div>`, `<button>` 등) |
| **attribute** | 요소의 속성 (`id`, `class`, `href` 등) |
| **text** | 요소 안의 텍스트 내용 |

### 2️⃣ HTML 요소 선택 & 조작

```jsx
// ✅ HTML 요소 선택
document.getElementById("btn");              // id로 1개 선택
document.getElementsByClassName("item");     // class로 여러 개 선택 (HTMLCollection)
document.querySelector(".box");              // CSS 선택자로 첫 번째 요소
document.querySelectorAll("li");             // CSS 선택자로 모두 선택 (NodeList)
```

```jsx
// ✅ HTML 요소 조작
const box = document.querySelector("#box");

box.style.backgroundColor = "red";          // 스타일 변경
box.innerHTML = "<strong>안녕!</strong>";    // HTML 포함 내용 변경
box.innerText = "텍스트만";                  // 텍스트만 변경
box.getAttribute("id");                      // 속성 값 읽기
box.setAttribute("class", "active");        // 속성 값 설정
box.classList.add("highlight");             // 클래스 추가
box.classList.remove("highlight");          // 클래스 제거
box.classList.toggle("hidden");             // 클래스 토글
```

### 3️⃣ 이벤트 핸들러 등록 3가지 방법

```jsx
// ❌ 방법 1: HTML 속성 방식 (비권장 — HTML과 JS가 뒤섞임)
// HTML과 자바스크립트가 섞여 유지보수가 어렵고, 복잡한 로직을 넣기 부적합
<button onclick="alert('클릭!')">버튼</button>
```

```jsx
// ⚠️ 방법 2: node 객체 프로퍼티 방식 (하나의 핸들러만 등록 가능)
const btn = document.querySelector("#btn");
btn.onclick = function () {
  console.log("클릭됨");
};
```

```jsx
// ✅ 방법 3: addEventListener (권장 — 여러 핸들러 등록 가능)
const btn = document.querySelector("#btn");
btn.addEventListener("click", function (event) {
  console.log("클릭됨!");
});

// 핸들러 제거
function handleClick(event) { console.log("클릭"); }
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick); // 반드시 동일한 함수 참조 필요
```

<aside>
🎓 <b>이벤트란?</b>
<p>"웹 브라우저가 알려주는 HTML 요소에 대한 사건의 발생"입니다. 마우스 이벤트(click, mousemove), 키보드 이벤트(keydown, keypress, keyup), 폼 이벤트(submit, change, focus, blur) 등이 있습니다. 이벤트가 발생하면 등록된 핸들러가 호출됩니다.</p>

### 4️⃣ 이벤트 객체 (event object)

이벤트 핸들러 함수의 **첫 번째 인수**로 자동으로 전달되는 객체입니다. 해당 이벤트에 대한 다양한 정보와 기능이 담겨 있습니다.

```jsx
// event.target: 이벤트가 실제로 발생한 요소에 접근
btn.addEventListener("click", function (event) {
  console.log(event.target);       // <button id="btn">...</button>
});

// event.clientX / clientY: 마우스 클릭 좌표
box.addEventListener("click", function (event) {
  console.log(event.clientX, event.clientY); // 예: 245 310
});

// event.key: 눌린 키 정보
window.addEventListener("keypress", function (event) {
  console.log(event.key.toUpperCase()); // 예: "A", "ENTER"
});

// event.preventDefault(): 브라우저 기본 동작 차단
const form = document.querySelector("#login-form");
form.addEventListener("submit", function (event) {
  event.preventDefault(); // form 제출 시 새로고침 방지
  console.log("아이디:", idInput.value);
});
```

### 5️⃣ 이벤트 Propagation (이벤트 전파)

이벤트가 발생하면, 해당 이벤트는 DOM 트리를 통해 **3단계로 전파**됩니다.

![이벤트 전파](image.png)

```
1. Capturing Phase: window → document → html → body → ... → 타겟 요소
2. Target Phase:   이벤트가 실제로 발생한 요소 (e.g. button#btn)
3. Bubbling Phase: 타겟 요소 → ... → body → html → document → window
```

```jsx
// 기본: Bubbling phase에서 이벤트 캐치 (세 번째 인자 없음 또는 false)
parent.addEventListener("click", function (event) {
  console.log("parent got event!");  // 자식 버튼 클릭 시에도 발생!
});

// Capturing phase에서 이벤트 캐치 (세 번째 인자 true)
parent.addEventListener("click", function (event) {
  console.log("capturing!");
}, true);
```

#### 대표 메서드 및 코드 예시

| **메서드** | **역할** | **주요 특징** |
| --- | --- | --- |
| **`stopPropagation()`** | 현재 이벤트가 상위(또는 하위) 요소로 **전파되는 것을 중단**시킵니다. | 같은 요소의 다른 이벤트 핸들러는 실행됩니다. |
| **`preventDefault()`** | 이벤트의 전파는 막지 않지만, 브라우저의 **기본 동작만 취소**합니다. (예: 링크 이동, 폼 제출) | 전파(버블링/캡처링)는 그대로 진행됩니다. |

```jsx
// stopPropagation(): 버블링 중단
document.querySelector('.child').addEventListener('click', (e) => {
  e.stopPropagation(); 
  console.log('자식 요소 클릭 (부모로 전파되지 않음)');
});

// preventDefault(): 기본 동작만 중단
document.querySelector('a').addEventListener('click', (e) => {
  e.preventDefault();
  console.log('링크 클릭은 감지되지만, 페이지 이동은 하지 않습니다.');
});
```

### 1️⃣ 타이머 함수

즉시 실행되는 일반 함수와 달리, 타이머 함수는 **함수 호출을 예약**합니다.

```jsx
// setTimeout: 특정 시간 후 1회 실행 (단위: ms)
const timeoutId = setTimeout(() => {
  console.log("3초가 지났습니다!");
}, 3000); // 3000ms = 3초

// 세 번째 인수 이후: 콜백에 전달할 인수
setTimeout((name) => console.log(`Hello, ${name}!`), 2000, "Elice");
// 출력: Hello, Elice! (2초 후)

// clearTimeout: timeout 취소
clearTimeout(timeoutId);
```

```jsx
// setInterval: 특정 시간마다 반복 실행
let count = 0;
const intervalId = setInterval(() => {
  count++;
  console.log(`Interval 실행: ${count}초 경과`);
  
  // clearInterval: interval 중지 (메모리 누수 방지에 중요!)
  if (count === 5) {
    clearInterval(intervalId);
    console.log("Interval 중지");
  }
}, 1000); // 1초마다
```

### 2️⃣ 실행 컨텍스트와 콜 스택

JS 엔진은 코드를 실행하기 전, **실행 컨텍스트**를 생성합니다.

```
실행 컨텍스트 생성 2단계:
  1. 생성 단계: 변수 선언문을 먼저 저장 (값 없음, undefined)
  2. 실행 단계: 나머지 코드 실행 + 변수 값 할당
  
  비유하자면,
  요리하기 전에 조리대에서 필요한 재료(변수) 꺼내놓고, 실행 단계에서 실제로 요리 완성하는 과정.
  요리(값 할당)하기 전이라, 1단계에서 변수 안에는 아무 것도 없다는 뜻으로 undefined.
```

생성되는 실행 컨텍스트는 모두 **콜 스택(Call Stack)**에서 관리됩니다. 스택은 LIFO(Last In, First Out) 구조로, 함수가 호출되면 쌓이고, 실행 완료 시 제거됩니다.

* **LIFO(Last In, First Out)**: 프링글스 통처럼 후입선출

```jsx
const x = 10;

function outerFunc() {
  const x = 20;
  function innerFunc() {
    console.log(x); // 20 (outerFunc의 x, 스코프 체인으로 탐색)
  }
  innerFunc(); // 콜 스택: 전역 → outerFunc → innerFunc
}

console.log(x); // 출력: 10
outerFunc(); // 콜 스택: 전역 → outerFunc
// 출력: 20

// 💡 콜 스택의 변화 과정
// 전역 시작 [전역] → outerFunc 실행 [전역, outer] → innerFunc 실행 [전역, outer, inner] → innerFunc 종료 [전역, outer] → outerFunc 종료 [전역] → 전체 종료 []
```

### 3️⃣ ES6 핵심 문법

**let, const (var 사용 지양)**

```jsx
let fruit = "오렌지";    // 값 변경 가능, 중복 선언 불가
fruit = "포도";         // ✅ OK
// let fruit = "레몬";  // ❌ Error

const banana = "바나나"; // 값 변경 불가(상수), 선언+초기화 동시에 필수
// banana = "딸기";     // ❌ Error

// var: 중복 선언 허용 → 의도치 않은 버그 유발 → 사용 지양
var x = 1;
var x = 2; // ✅ 에러 없음 (문제!)
console.log(x); // "2" (기존 데이터가 날아감)
```

**스코프(Scope): 변수를 참조할 수 있는 유효 범위**

```jsx
// var: 함수 레벨 스코프 (블록 { }을 무시)
if (true) {
  var leaked = "나는 if 밖에서도 보여요";
}
console.log(leaked); // "나는 if 밖에서도 보여요" (의도치 않은 접근!)

// let/const: 블록 레벨 스코프 (블록 { } 안에서만 유효)
if (true) {
  let safe = "나는 if 안에서만 유효해요";
}
// console.log(safe); // ❌ ReferenceError
```

**화살표 함수, 구조 분해 할당, Spread, 배열 메서드**

```jsx
// 화살표 함수
const add = (a, b) => a + b;

// 구조 분해 할당
const { name, age } = { name: "김민준", age: 20 };
const [first, second] = ["사과", "바나나"];

// Spread 문법
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

// 배열 메서드 (map, filter, reduce)
const nums = [1, 2, 3, 4, 5];
const doubled = nums.map((n) => n * 2);         // [2, 4, 6, 8, 10]
const evens = nums.filter((n) => n % 2 === 0);  // [2, 4]
const sum = nums.reduce((acc, n) => acc + n, 0); // 15
```
