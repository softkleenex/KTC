### 변수 익히기

- 변수의 정의
	- 데이터를 빈 상자
	- 변수의 선언: 빈 상자에 이름표를 붙히는 것
	- 이름을 통해 데이털르 사용할 수 있다.
	- 데이터가 재할당되면 기존에 있던 데이터는 사라진다.
- 변수의 선언
	- 변수명과 매핑하고자 하는 데이터의 메모리 주소를 저장할 또하나의 메모리 공간을 확보하는것.
	- let, const, var 3개의 키워드를 사용한다
- let을 사용한 변수의 선언
	- var과는 다르게 let은 변수의 재 선언이 불가능하다. 
	- 재할당은 가능
	- 즉, 데이터의 할당이 변경 될 수 있을때 사용
- const
	- var과는 다르게 let 키워드는 변수의 재 선언이 불간ㅇ하다
	- const 키워드는 데이터의 재할당이 불가능하다
	- const가 let과 다른점이있다면, 반드시 선언과 초기화를 동시에 징행되어야한다.
	- 즉, 데이터의 할당이 변경되지않을때 사용
- var
	- 변수 중복선언이 가능하여, 예기치 못하 값을 반환할 수 있다.
	- 함수 레벨 스코프로 인해 함수 외부에서 선언한 변수는 모두 전역 변수로 된다.
	- 변수 선언문 이전에 변수를 참조하면 언제나 undefined를 반환한다.
	- 즉, Es6 이전 버전에서 사용

- 변수의 작명
	- 구제적이고, 연관성 있는 저장된 값을 잘 알수있는 이름이 좋다.
	- 예약어를 사용하지말자
	- 변수의 단어가2개 이상일 시에는 첫 글자가 대문자로 시작되는 camelcase를 사용하는것이 좋다.
	- 숫자로 시작하지않는다
	- 특수 문자를 금지합니다.(_, $ 제외)

### 데이터타입-객체 타입

- 객체 타입
	- 자바스크립트의 기본타입을 제외한 모든 값은 객체다.
	- 객체는 이름(key): 값(value)의 형태로 프로퍼티들을 저장하는 컨테이너.
	- 기본타입으로 분류되는 숫자, 문자열, 불린 값, undefined, null 값을 제외한 모든 값은 객체이다.
	- object {key: value}
		- 다수의 property를 하나의 이름으로 묶어놓은 집합체
- ```
	var emptyObject = {};
	console.log(typeof emptyObject); // emptyObject)
	
	var person = {
	  name: 'elice',
	  gender: 'male', 
	  sayHello: function() { 
		console.log('Hi! My name is ' + this.name); 
	  }
	}; 
	
	person.sayHello();
	```
- array \['text',0,function\()\{},Boolean,null]: 순서가 있는 데이터의 집합
	- 리스트와 비슷한 객체로서 순회와 변형 작업을 수행하는 메서드를 갖습니다.
	- JavaScrip 배열은 길이도, 각 요소의 자료형도 고정되어있지 않습니다.
	- Object와 달리 배열은 인덱스가 0으로 시작하는 숫자형 인덱스를 사용하며 한개의 변수의 여러 개의 값을 순차적으로 저장할 수 있다.
	- ```
	var myArray1 = new Array(); // 길이가 0인 예전 방식의 배열 생성
	var myArray2 = new Array(10); // 길이가 10인 예전 방식의 배열 생성
	var myArray3 = []; // 길이가 0인 literal notation 방식의 배열 생성
	var myArray4 = new Array(1, 2, 3, "elice", "cat"); // 예전 방식의 생성과 동시에 초기화
	var myArray5 = [1, 2, 3, "elice", "cat"]; // literal notation 방식의 생성과 동시에 초기화
	console.log(myArray1);
	  ```
- function: 일련의 작업을 목적으로 수행에 설계된 독립적 블록
	- Javascript의 함수는 객체타입을 갖는1급 객체
	- 변수나, 데이터를 구조안에 담을 수 있다.
	- 다른 함수에 파라미터(매개변수로 전달할 수 있다)
	- 리턴값을 사용할 수 있다.
	- 함수는 return 문을 포함할 수 있습니다.
	- 이런 반환문을 통해 호출자는 함수에서 실행된 결과를 전달 받을 수 있습니다.
	- ```
	  function addNum(x, y)
	  {
	  return x + y;
	  }
	  console.log(addNum(2, 3))
	  ```  
- 화살표 함수
	- ES6부터 새롭게 등장한 함수 작성 방식
	- funciton 키워드 대신 =>(화살표) 기호를 사용한다. 같은 동작을 더 짧고 간결하게 표현할 수 있다.
	- 코드가 짧고 가독성이 좋다.
	- 한줄짜리 함수는 return도 생략 가능하다.
	- ```
	  const addNum = (x, y) => x + y;
		```
- Null 명시적으로 비어있음을 표시할 때
	- 의도적으로 ㅁ변수에 값이 없다는것을 명시할 떄 사용한다.
	- 변수가 기억하는 메모리 어드레스의 참조 정보를 제거하는것을 의미하며 자바스크립트 엔진은 누구도 참조하지않는 메모리 영역에 대해 가비지 콜렉션을 수행함.
	- typeof 에서 null객체의 자료형을 확인할 때 object(객체)를 반환하는데, null이 빈 참조를 나타내는 데 자주 사용되기 때문이다. 그래서 원시 자료형으로 생각해도되지만 엄밀히 말해서는 null은 객체이고 참조 자료형이다.
	- ```
	  var nullTyle = null;

### JavaScript의 다양한 연산

- 산술연산
	- 우리가 알고있는 기본적인 사칙연산
- 증감연산
	- 1씩 증가. ++(Increment)
	- 1씩 감소. --(Decrement)
	- 후위연산
	  연산자를  변수의 뒤에 위치 시킨다. 변수에 먼저 할당(대입)한 뒤에 연산.
	- 전위연산
	  연산자를  변수의 앞에 위치 시킨다. 연산을 먼저 한 뒤에 변수에 할당(대입).
- 복합 대입 연산
  대입 연산자는 변수에 값을 대입할 떄 사용하는 이항 연산자, 피 연산자들의 결합 방향은 오른쪽에서 왼쪽이다. 또한, 앞서 살펴본 산줄 연산자와 다양한 복합 대입 연산자가 존재한다.
	- += 증가 대입
	- -= 감소 대입
	- *= 곱 대입
- 비교 연산
  변수의 값을 어떤 값 또는 다른 변수와 비교할 때 사용하는것이 비교 연산자(Comparison Operator)
- |**연산자**|**설명**|
|---|---|
|`a = b`|A에 b를 대입한다.|
|`a == b`|A와 b는 같다.|
|`a === b`|A와 B는 data type도 같다.|
|`a != b`|A와 B는 같지 않다.|
|`a > b`|A는 b보다 크다.|
|`a < b`|A는 b보다 작다.|
|`a >= b`|A는 b보다 크거나 같다.|
|`a <= b`|A는 b보다 작거나 같다.|
	- == 연산자는 타입일 자동으로 맞춰주기 때문에, 아래의 연산이 true가 나오게 된다, 따라서 === 권장
		- '' == false
		- '0' -= 0
		- null == undefined
- 논리 연산
	- &&(and) ||(or) !(not)

### 조건문


**기본 원리**
- 일정한 조건이 true로 평가되면 코드블록이 실행됩니다.
- 조건문은 그 밖의 다른 조건 else if를 가질 수 있습니다.
- 조건문의 else 그 밖의 모든 조건을 갖을 수 있습니다.
** Truthy & Falsy**
- boolean이 아닌 값을 넣어도(숫자 1, 0) JS는 알아서 t, f로 해석한다. 이떄 f로 취급되는 값들을 Falsy,  Truthy이다.
>  주의: 빈배열\[], 빈 객체 \{}는 Truthy이다.

**문법 구조**
- if(조건문){실행문}
  표현식을 평가하고 그 결과값을 boolean값으로 반환한다.
- if(조건문){실행문}else if{실행문}
  첫번쨰 조건문이 거짓이고, 다음 조건이 참일 떄 코드 블럭을 진행합니다.
- 3항 연산자: `(조건문)? True 실행문 : false실행문`
