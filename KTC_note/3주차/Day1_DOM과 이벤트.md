### DOM이란?


**DOM** : 문서 객체 모델(Document Objext Model) 구조화된 문서(XML, HTML)을 객체 형태로 표현하는방식
- 자바스크립트 엔진이 HTML 코드를 해석한 결과. 각각의 태그가 자바스크립트의 node객체로 변환된다

**노드**: HTML DOM 에서 정보를 저장하는 계층적 단위 DOM은 총 12가지 노드 유형으로 구성되지만, 개발에서는 document, element, attribute, text 객체가 핵심적 역할
**노드 트리**: 노드들의 집합으로, 노드간의 관계를 트리 구조로 나타낸 것

노드 트리의 구성원
document: 웹 페이지
element: 개별 HTML 요소
	이것들을 이용해서 HTML 요소의 선택과 조작. HTML 요소의 생성 및 추가. HTML 이벤트 핸들러 추가 가능!

### 02_이벤트

- **이벤트**: 웹 브라우저가 알려주는 HTML 요소에 대한 사건의 발생, 자바스크립트는 발생 이벤트에 반응하여 특정 동작을 수행할 수 있다.
	- 마우스 이벤트
	- 키 이벤트
	- 폼 이벤트

이벤트가 발생하면 작동할 함수, 핸들러를 추가하는 방식

|**사용 예시**|**설명**|
|---|---|
|`<요소 onclick="handler()" />`<br><br>  <br><br>`function handler() { ... }`|요소의 여러 이벤트에 대해 이벤트 핸들러 연결|
|`요소.onclick = function() { ... }`<br><br>  <br><br>`요소.onsubmit = function() { ... }`<br><br>  <br><br>`...`|〃|
|`요소.addEventListener("click", function() { ... })`|〃|

### 이벤트 핸들러

**이벤트 핸들러**: 이벤트가 발생했을 떄 처리를 담당하는 함수. 지정된 이벤트가 발생하면, 그 요소에 등록된 이벤트 핸들러를 실행한다

- 추가 방법
	- HTML 요소의 'on이벤트명' 속성에 동작 추가
		- HTML 코드에 JS 코드가 포함되는 방식으로, 권장되지않는다.
	- node 객체의 'on이벤트명' 프로퍼티에 핸들러 함수를 등록하는방법
	- 첫번째 인수로 '이벤트명' 전달,(on을 붙이지 않는다). 두번쨰 인수로 핸들러 함수 전달
		- 가장 권장되는 함수.

---
### 이벤트 객체

**event 객체**: 이벤트 핸들러 함수에 첫번째 인수로 전달되는 객체 > 해당 이벤트에 대한 정보와 기능이 담겨있다.

**키보드 관련 이벤트**: event객체를 통해 키보드에 대한 정보를 얻을 수 있다(누른 키 정보 등). 이처럼 각 이벤트 발생에 대한 정보를 event 객체를 통해 얻을 수 있다.

**target**: event.target 프로펕티를 통해 이벤트가 발생한 타겟 요소의 node객체에 접근할 수 있다.

**preventDefalut**: form 태그에서 submit 버튼의 기본 동작은 form 제출시에 새로고침. 이 기본동작을 막기 위해 event 객체의 event.preventDefault() 메서드를 사용한다.

**removeEventListener**: 메서드로 등록된 이벤트 핸들러 제거

### 이벤트 Propagation

DOM에서 버튼 요소는 3개의 부모 요소와 그 상위 노드인 \<body>, \<html>, document, window 등이 있다.
가장 하위 요소인 버튼을 클릭하면. 이 이벤트는 하위요소 뿐만이 아니라,  DOM트리를 통해서 전파되어 모든 요소에서 발생한다.

1. Capturing Phase
	- **방향:** 최상위 부모 요소(`window`) ➔ 실제 이벤트가 발생한 **타겟 요소** (위에서 아래로)
	- **설명:** 이벤트가 자식 요소를 향해 DOM 트리를 따라 내려가는 단계입니다.
	- **특징:** 실제 실무나 일반적인 이벤트 리스너(`addEventListener`)에서는 기본적으로 이 단계를 감지하지 않고 건너뜁니다.
	    - 만약 캡처링 단계의 이벤트를 감지하고 싶다면 `addEventListener('click', 함수, true)` 처럼 3번째 인자에 `true` 또는 `{ capture: true }` 옵션을 주어야 합니다._
2. Target Phase
- **방향:** 이벤트가 실제 발생한 **타겟 요소에 도달**한 상태
- **설명:** 이벤트가 목적지에 도착하여, 해당 요소에 직접 등록된 이벤트 핸들러가 실행되는 단계입니다
3. Bubbling Phase
- **방향:** 실제 이벤트가 발생한 **타겟 요소** ➔ 최상위 부모 요소(`window`) (아래에서 위로)
- **설명:** 마치 물속의 공기방울(Bubble)이 수면 위로 올라가는 것처럼, 이벤트가 다시 부모 요소를 타고 조상들에게로 거슬러 올라가는 단계입니다.
- **특징:** 자바스크립트의 **기본 이벤트 전파 방식**입니다. 우리가 평소에 쓰는 대부분의 이벤트는 이 버블링 단계에서 실행되고 감지됩니다.


버튼 클릭시에 버블링으로 인해서 의도하지않은 부모의 핸들러까지도 실행될수도 있는데,,, 어떻게 하지?

**event.stopPropagation()**: 해당 요소의 핸들러는 실행되지만, 이후 부모로의 전파는 중단된다.
- preventDefault(브라우저의 기본 동작을 막음)와 헷갈리지 말 것.
